Practicum #2 – Double Ended Queue

Introduction

A queue is a standard data structure supporting first-in, first-out access to stored data. In a queue, items are added at the back end and removed from the front.

A double ended queue (or dequeue) is a data structure where items can be added to or removed from either end. For this practicum you will incrementally implement a dequeue data structure as a singly linked list of nodes, where each node contains a string item.

The Problem

The basic operations to manipulate a dequeue of strings are the following. The parameter deqhead is passed  in as either a pointer to the first element (node)  of the queue, or as  the address of where the pointer to the first element  is being stored.

void deq_put_head(node **deqhead, char *value)

Puts a node containing a copy of the string value at the head (front) of the dequeue. Entires are not checked for uniqueness.

void deq_put_tail(node **deqhead, char *value)

Puts a node containing a copy of the string value at the tail (back end) of the dequeue. Entires are not checked for uniqueness.

void deq_get_head(node **deqhead, char *value)

If the dequeue is empty, simply returns.

Otherwise, removes the node at the head of the dequeue (adjusting the dequeue pointers as necessary), copies the stored string to value, and frees all the storage associated with the node.

void deq_get_tail(node **deqhead, char *value)

If the dequeue is empty, simply returns.

Otherwise, removes the node at the tail of the dequeue (adjusting the dequeue pointers as necessary), copies the stored string to value, and frees all the storage associated with the node.

int deq_size(node *deqhead)

Returns the number of items in the dequeue. In our implementation this is done by traversing the dequeue list from head to tail, counting the number of nodes as we go.

bool deq_is_empty(node *deqhead)

Returns true if the dequeue has no elements (size is zero), false otherwise.
